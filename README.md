# 沪深300价值股票量化选股系统

基于**静态价值型烟蒂股**投资策略，从沪深300成分股中量化筛选符合价值投资标准的股票。

## 策略核心：静态烟蒂股三要义

本系统将"史诗级韭菜"空间中定义的烟蒂股选股原则量化为可执行代码，核心逻辑围绕三大支柱：

1. **存量资产垫**（40%权重）——企业净可变现资产相较于市值具有明显溢价，按T0/T1/T2三级分级
2. **低维持营运开支**（30%权重）——自由现金流为正，资本开支率低于50%，不消耗存量资产
3. **资产兑现逻辑**（30%权重）——具备分红、盈利恢复或事件驱动等明确的价值实现路径

### 资产垫分级标准

| 等级 | 条件 | 含义 |
|------|------|------|
| T0 | 保守比率 ≥ 0.8 且 宽松比率 ≥ 1.5 | 最强资产垫，接近清算套利 |
| T1 | 宽松比率 ≥ 1.0 且 保守现金净额 > 0 | 强资产垫，现金扣有息负债后超市值 |
| T2 | 宽松现金净额 > 0 且 流动资产净值比率 > 0.5 | 中等资产垫，流动资产提供支撑 |
| FAIL | 不满足以上条件 | 资产垫不足，不入选 |

## 项目结构

```
├── main.py                  # 主程序入口
├── config.yaml              # 用户可调配置文件
├── requirements.txt         # Python依赖
├── common/                  # 公共模块（异常、数据模型、重试、日志）
├── config/                  # 配置管理（YAML配置、Settings单例）
├── fetcher/                 # 数据获取层（成分股、财报、市值、分红）

├── docs/                    # Markdown格式文档
├── output_data/results/     # 选股结果CSV输出
└── logs/                    # 运行日志
```

## 快速开始

### 环境要求

- Python 3.10+
- Windows / Anaconda

### 安装

```bash
pip install -r requirements.txt
```

### 运行

```bash
python main.py
```

### 输出

- 终端：Rich格式化表格，按综合评分排序
- CSV：`output_data/results/stock_selection_YYYYMMDD_HHMMSS.csv`
- 日志：`logs/stock_selection_YYYYMMDD.log`

## 配置说明

所有选股指标阈值均可通过 `config.yaml` 调整，无需修改代码：

- `asset_cushion`：资产垫分级阈值（T0/T1/T2各条件比率）
- `operation_safety`：经营安全阈值（自由现金流、资本开支率）
- `redemption_safety`：兑现安全阈值（分红年数、股息率、盈利恢复回看年数）
- `scoring`：评分权重与分值
- `delisting`：退市风险过滤参数

修改配置后直接运行即可生效。

## 数据源

| 数据类型 | 主数据源 | 备选 |
|----------|----------|------|
| 沪深300成分股 | akshare (中证指数) | - |
| 财务报表 | 新浪财经 | - |
| 市值行情 | 腾讯行情API | akshare (东方财富) |
| 分红数据 | akshare | - |

数据自动缓存至 `output_data/cache/`，财报数据缓存4小时，行情数据缓存1小时。

## 退市风险过滤

系统在筛选前自动剔除以下股票：
- 名称含 ST / *ST 标记
- 扣非净利润为负且营业收入 < 1亿元
- 期末净资产为负
