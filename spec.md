# 需求规格文档：沪深300价值股票量化选股系统

## 1. 项目概述

### 1.1 背景
本项目（"史诗级韭菜"知识空间）聚焦于A股和H股的深度价值挖掘，核心策略为**静态价值型烟蒂股**投资——即以当前可变现资产为主要安全垫，寻找市场价格明显低于真实价值的证券。现需将该策略量化为可执行的代码，从沪深300成分股中筛选符合标准的股票。

### 1.2 目标
编写Python量化代码，基于项目定义的"静态价值型烟蒂股三要义"（存量资产垫、低维持营运开支、资产兑现逻辑），从沪深300指数成分股中自动选取符合价值投资标准的股票代码，并输出分级筛选结果。

### 1.3 范围
- 股票池：沪深300成分股
- 市场范围：A股（上海证券交易所、深圳证券交易所）
- 策略类型：静态价值型烟蒂股（非成长型、非动态价值型）

---

## 2. 功能需求（EARS格式）

### 2.1 沪深300成分股获取

**REQ-001**: 当系统启动选股流程时，系统应从数据源获取当前沪深300指数的全部成分股代码及名称。

- 数据源优先级：akshare > tushare > baostock
- 返回字段：股票代码、股票名称、所属交易所

### 2.2 财务数据采集

**REQ-002**: 对于每只沪深300成分股，系统应获取其最近一期财报的以下财务数据：
- 资产负债表：现金及现金等价物、流动资产合计、总资产、流动负债合计、总负债、有息负债（短期借款+长期借款+应付债券）、净资产（所有者权益合计）
- 利润表：营业收入、营业成本、净利润、扣非净利润
- 现金流量表：经营活动现金流净额、资本开支（购建固定资产无形资产等支付的现金）、自由现金流（经营现金流-资本开支）

**REQ-003**: 系统应获取每只股票的市值数据，包括：总市值、流通市值、最新股价。

**REQ-004**: 系统应获取每只股票的分红数据，包括：近3年累计现金分红金额、股息率。

### 2.3 核心筛选：存量资产垫（第一要义）

**REQ-005**: 系统应计算每只股票的"资产垫"指标，定义如下：
- `现金净额（保守）= 现金及现金等价物 - 总负债`
- `现金净额（宽松）= 现金及现金等价物 - 有息负债`
- `资产垫比率（保守）= 现金净额（保守） / 总市值`
- `资产垫比率（宽松）= 现金净额（宽松） / 总市值`

**REQ-006**: 系统应按资产垫等级将股票分级：

| 等级 | 条件 | 说明 |
|------|------|------|
| T0 | 现金净额（保守）≥ 总市值 × 0.8 且 现金净额（宽松）≥ 总市值 × 1.5 | 最强资产垫，接近清算价值≥市值 |
| T1 | 现金净额（宽松）≥ 总市值 且 现金净额（保守）> 0 | 强资产垫，现金扣除有息负债后仍超市值 |
| T2 | 现金净额（宽松）> 0 且 流动资产 - 总负债 > 总市值 × 0.5 | 中等资产垫，流动资产提供支撑 |
| 不符合 | 不满足以上任何条件 | 资产垫不足，不入选 |

### 2.4 核心筛选：低维持营运开支（第二要义）

**REQ-007**: 系统应计算每只股票的"经营安全"指标：
- `自由现金流 = 经营活动现金流净额 - 资本开支`
- `资本开支率 = 资本开支 / 经营活动现金流净额`（衡量维持运营所需再投入比例）

**REQ-008**: 系统应筛选经营安全条件：
- 自由现金流 > 0（经营不消耗存量资产）
- 资本开支率 < 0.5（维持运营所需再投入低于经营现金流的50%）

### 2.5 核心筛选：资产兑现逻辑（第三要义）

**REQ-009**: 系统应评估每只股票的"兑现安全"指标，满足以下任一条件即视为具有兑现逻辑：
- 近3年持续分红且平均股息率 > 2%（分红兑现路径）
- 近1年存在资产处置、股权回购等特殊事件（事件兑现路径）
- 近3年扣非净利润为正且呈恢复趋势（盈利恢复兑现路径）

**REQ-010**: 对于盈利恢复兑现路径，系统应判断：近3年扣非净利润均为正，且最近一年扣非净利润 > 前一年扣非净利润。

### 2.6 综合筛选与评分

**REQ-011**: 系统应将三要义综合评分，对每只股票计算价值评分：
- 资产垫得分（40%权重）：根据T0/T1/T2等级分别赋分100/70/40，不符合为0
- 经营安全得分（30%权重）：自由现金流>0且资本开支率<0.5得100，部分满足得50，不满足得0
- 兑现安全得分（30%权重）：满足兑现逻辑得100，不满足得0

**REQ-012**: 系统应按综合得分从高到低排序，输出最终选股结果。

### 2.7 退市风险过滤

**REQ-013**: 系统应在筛选前先进行退市风险过滤，剔除具有以下特征的股票：
- 股票简称包含"ST"或"*ST"（已被实施风险警示）
- 最近一年扣非净利润为负且营业收入 < 1亿元（触及财务类退市风险线）
- 期末净资产为负值

### 2.8 结果输出

**REQ-014**: 系统应输出选股结果，包含以下信息：
- 股票代码、股票名称
- 资产垫等级（T0/T1/T2）、资产垫比率
- 自由现金流、资本开支率
- 兑现逻辑类型
- 综合评分
- 排名

**REQ-015**: 系统应支持将结果输出为CSV文件和终端表格两种格式。

## 3. 非功能需求

**NFR-001**: 系统单次完整选股流程（获取数据+筛选+输出）应在5分钟内完成。

**NFR-002**: 系统应具备数据获取失败的重试机制，单个数据源最多重试3次，间隔5秒。

**NFR-003**: 系统应记录运行日志，包括数据获取情况、筛选过程、异常信息。

**NFR-004**: 代码应模块化设计，选股指标阈值可通过配置文件调整，无需修改代码。

**NFR-005**: 系统应处理数据缺失情况，对缺少关键财务数据的股票标注"数据不完整"并跳过，不中断整体流程。

---

## 4. 约束条件

**CON-001**: 编程语言：Python 3.10+
**CON-002**: 数据源库：优先使用akshare（免费开源），备选tushare（需token）、baostock
**CON-003**: 运行环境：Windows + Anaconda
**CON-004**: 不使用任何付费数据接口
**CON-005**: 不进行自动交易，仅输出选股结果供人工决策参考

## 6. 选股指标汇总

| 指标类别 | 指标名称 | 计算方式 | 筛选标准 |
|----------|----------|----------|----------|
| 资产垫 | 现金净额（保守） | 现金及等价物 - 总负债 | ≥ 市值×0.8（T0级） |
| 资产垫 | 现金净额（宽松） | 现金及等价物 - 有息负债 | ≥ 市值×1.5（T0级），≥ 市值（T1级） |
| 资产垫 | 流动资产净值 | 流动资产 - 总负债 | > 市值×0.5（T2级） |
| 经营安全 | 自由现金流 | 经营现金流 - 资本开支 | > 0 |
| 经营安全 | 资本开支率 | 资本开支 / 经营现金流 | < 0.5 |
| 兑现安全 | 股息率 | 近3年均分红 / 当前股价 | > 2% |
| 兑现安全 | 盈利恢复 | 近3年扣非净利润趋势 | 为正且递增 |
| 风控 | 退市风险 | ST标记/财务指标 | 剔除风险股 |

## 7. 术语定义

| 术语 | 定义 |
|------|------|
| 烟蒂股 | Cigar Butt Stock，市场价格远低于可变现资产价值的股票 |
| 静态价值 | 基于当前资产负债表的存量价值，非未来成长预期 |
| 资产垫 | 净可变现资产超出市值的溢价部分，提供安全边际 |
| 有息负债 | 需支付利息的债务，包括短期借款、长期借款、应付债券 |
| T0/T1/T2 | 资产垫等级，T0最强（接近清算套利），T2最弱 |
| 自由现金流 | 经营活动产生的现金流扣除资本开支后的剩余 |
| 资本开支率 | 维持运营所需再投入占经营现金流的比例 |
