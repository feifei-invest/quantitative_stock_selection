# 需求规格文档：沪深300价值股票量化选股系统

## 1. 项目概述

### 1.1 背景
本项目（"史诗级韭菜"知识空间）聚焦于A股和H股的深度价值挖掘，核心策略为**静态价值型烟蒂股**投资——即以当前可变现资产为主要安全垫，寻找市场价格明显低于真实价值的证券。现需将该策略量化为可执行的代码，从沪深300成分股中筛选符合标准的股票。

### 1.2 目标
编写Python量化代码，基于项目定义的"静态价值型烟蒂股三要义"（存量资产垫、低维持营运开支、资产兑现逻辑），从沪深300指数成分股中自动选取符合价值投资标准的股票代码，并输出分级筛选结果。

### 1.3 范围
- 股票池：沪深300成分股
- 市场范围：A股（上海证券交易所、深圳证券交易所）
- 策略类型：静态价值型烟蒂股（非成长型、非动态价值型）

---

## 2. 功能需求（EARS格式）

### 2.1 沪深300成分股获取

**REQ-001**: 当系统启动选股流程时，系统应从数据源获取当前沪深300指数的全部成分股代码及名称。

- 数据源优先级：akshare > tushare > baostock
- 返回字段：股票代码、股票名称、所属交易所

### 2.2 财务数据采集

**REQ-002**: 对于每只沪深300成分股，系统应获取其最近一期财报的以下财务数据：
- 资产负债表：现金及现金等价物、流动资产合计、总资产、流动负债合计、总负债、有息负债（短期借款+长期借款+应付债券）、净资产（所有者权益合计）
- 利润表：营业收入、营业成本、净利润、扣非净利润
- 现金流量表：经营活动现金流净额、资本开支（购建固定资产无形资产等支付的现金）、自由现金流（经营现金流-资本开支）

**REQ-003**: 系统应获取每只股票的市值数据，包括：总市值、流通市值、最新股价。

**REQ-004**: 系统应获取每只股票的分红数据，包括：近3年累计现金分红金额、股息率。

### 2.3 核心筛选：存量资产垫（第一要义）

**REQ-005**: 系统应计算每只股票的"资产垫"指标，定义如下：
- `现金净额（保守）= 现金及现金等价物 - 总负债`
- `现金净额（宽松）= 现金及现金等价物 - 有息负债`
- `资产垫比率（保守）= 现金净额（保守） / 总市值`
- `资产垫比率（宽松）= 现金净额（宽松） / 总市值`

**REQ-006**: 系统应按资产垫等级将股票分级：

| 等级 | 条件 | 说明 |
|------|------|------|
| T0 | 现金净额（保守）≥ 总市值 × 0.8 且 现金净额（宽松）≥ 总市值 × 1.5 | 最强资产垫，接近清算价值≥市值 |
| T1 | 现金净额（宽松）≥ 总市值 且 现金净额（保守）> 0 | 强资产垫，现金扣除有息负债后仍超市值 |
| T2 | 现金净额（宽松）> 0 且 流动资产 - 总负债 > 总市值 × 0.5 | 中等资产垫，流动资产提供支撑 |
| 不符合 | 不满足以上任何条件 | 资产垫不足，不入选 |

### 2.4 核心筛选：低维持营运开支（第二要义）

**REQ-007**: 系统应计算每只股票的"经营安全"指标：
- `自由现金流 = 经营活动现金流净额 - 资本开支`
- `资本开支率 = 资本开支 / 经营活动现金流净额`（衡量维持运营所需再投入比例）

**REQ-008**: 系统应筛选经营安全条件：
- 自由现金流 > 0（经营不消耗存量资产）
- 资本开支率 < 0.5（维持运营所需再投入低于经营现金流的50%）

### 2.5 核心筛选：资产兑现逻辑（第三要义）

**REQ-009**: 系统应评估每只股票的"兑现安全"指标，满足以下任一条件即视为具有兑现逻辑：
- 近3年持续分红且平均股息率 > 2%（分红兑现路径）
- 近1年存在资产处置、股权回购等特殊事件（事件兑现路径）
- 近3年扣非净利润为正且呈恢复趋势（盈利恢复兑现路径）

**REQ-010**: 对于盈利恢复兑现路径，系统应判断：近3年扣非净利润均为正，且最近一年扣非净利润 > 前一年扣非净利润。

### 2.6 综合筛选与评分

**REQ-011**: 系统应将三要义综合评分，对每只股票计算价值评分：
- 资产垫得分（40%权重）：根据T0/T1/T2等级分别赋分100/70/40，不符合为0
- 经营安全得分（30%权重）：自由现金流>0且资本开支率<0.5得100，部分满足得50，不满足得0
- 兑现安全得分（30%权重）：满足兑现逻辑得100，不满足得0

**REQ-012**: 系统应按综合得分从高到低排序，输出最终选股结果。

### 2.7 退市风险过滤

**REQ-013**: 系统应在筛选前先进行退市风险过滤，剔除具有以下特征的股票：
- 股票简称包含"ST"或"*ST"（已被实施风险警示）
- 最近一年扣非净利润为负且营业收入 < 1亿元（触及财务类退市风险线）
- 期末净资产为负值

### 2.8 结果输出

**REQ-014**: 系统应输出选股结果，包含以下信息：
- 股票代码、股票名称
- 资产垫等级（T0/T1/T2）、资产垫比率
- 自由现金流、资本开支率
- 兑现逻辑类型
- 综合评分
- 排名

**REQ-015**: 系统应支持将结果输出为CSV文件和终端表格两种格式。

## 3. 非功能需求

**NFR-001**: 系统单次完整选股流程（获取数据+筛选+输出）应在5分钟内完成。

**NFR-002**: 系统应具备数据获取失败的重试机制，单个数据源最多重试3次，间隔5秒。

**NFR-003**: 系统应记录运行日志，包括数据获取情况、筛选过程、异常信息。

**NFR-004**: 代码应模块化设计，选股指标阈值可通过配置文件调整，无需修改代码。

**NFR-005**: 系统应处理数据缺失情况，对缺少关键财务数据的股票标注"数据不完整"并跳过，不中断整体流程。

## 6. 选股指标汇总

| 指标类别 | 指标名称 | 计算方式 | 筛选标准 |
|----------|----------|----------|----------|
| 资产垫 | 现金净额（保守） | 现金及等价物 - 总负债 | ≥ 市值×0.8（T0级） |
| 资产垫 | 现金净额（宽松） | 现金及等价物 - 有息负债 | ≥ 市值×1.5（T0级），≥ 市值（T1级） |
| 资产垫 | 流动资产净值 | 流动资产 - 总负债 | > 市值×0.5（T2级） |
| 经营安全 | 自由现金流 | 经营现金流 - 资本开支 | > 0 |
| 经营安全 | 资本开支率 | 资本开支 / 经营现金流 | < 0.5 |
| 兑现安全 | 股息率 | 近3年均分红 / 当前股价 | > 2% |
| 兑现安全 | 盈利恢复 | 近3年扣非净利润趋势 | 为正且递增 |
| 风控 | 退市风险 | ST标记/财务指标 | 剔除风险股 |

## 7. 术语定义

| 术语 | 定义 |
|------|------|
| 烟蒂股 | Cigar Butt Stock，市场价格远低于可变现资产价值的股票 |
| 静态价值 | 基于当前资产负债表的存量价值，非未来成长预期 |
| 资产垫 | 净可变现资产超出市值的溢价部分，提供安全边际 |
| 有息负债 | 需支付利息的债务，包括短期借款、长期借款、应付债券 |
| T0/T1/T2 | 资产垫等级，T0最强（接近清算套利），T2最弱 |
| 自由现金流 | 经营活动产生的现金流扣除资本开支后的剩余 |
| 资本开支率 | 维持运营所需再投入占经营现金流的比例 |

# 港股通选股功能需求规格（EARS格式）

## 8. 港股通选股功能概述

### 8.1 背景
在现有沪深300价值股票量化选股系统基础上，扩展支持港股通（沪港通+深港通）标的股票的选股功能。选股策略沿用"静态价值型烟蒂股三要义"（存量资产垫、低维持营运开支、资产兑现逻辑），但需针对港股市场的数据源差异、会计准则差异、货币单位差异、退市规则差异等进行适配。

### 8.2 目标
扩展现有量化选股系统，使其能够从港股通标的股票中，按照相同的烟蒂股三要义逻辑筛选符合价值投资标准的港股，并输出分级筛选结果。

### 8.3 范围
- 股票池：港股通标的股票（沪港通+深港通可买卖股票）
- 市场范围：香港联合交易所（港交所）主板
- 策略类型：静态价值型烟蒂股（与A股同一策略逻辑）
- 货币单位：港币（HKD）
- 会计准则：IFRS / HKFRS（香港财务报告准则）

### 8.4 与A股模块的复用关系
- **完全复用**：资产垫分级逻辑（T0/T1/T2）、经营安全评估逻辑、兑现安全评估逻辑、综合评分与排序逻辑、结果输出格式
- **需适配**：成分股获取、财务数据采集（字段映射）、市值数据获取、分红数据获取、退市风险过滤规则
- **需新增**：港股通成分股获取模块、港股财务数据字段映射配置、港交所退市规则过滤、汇率换算支持

## 9. 港股通选股功能需求（EARS格式）

### 9.1 港股通成分股获取

**REQ-HK-001**: When 系统以港股通模式启动选股流程时，the 系统应从数据源获取当前港股通（沪港通+深港通）的全部标的股票代码及名称。

- 数据源：akshare的港股通标的接口（如 `stock_hk_ggt_components_em`）
- 返回字段：股票代码（5位港股代码，如"00700"）、股票名称、所属板块
- 港股代码格式：5位数字，不足5位前补零

**REQ-HK-002**: When 港股通成分股获取成功时，the 系统应记录沪港通标的数量和深港通标的数量，并合并去重为最终的港股通股票池。

- 验收条件：获取港股通成分股 → 返回列表中无重复代码，且包含沪港通和深港通的全部标的

**REQ-HK-003**: If 港股通成分股获取失败时，the 系统应记录错误日志并终止选股流程，不使用空列表继续后续步骤。

### 9.2 港股财务数据采集与字段映射

**REQ-HK-004**: For 每只港股通标的股票，the 系统应获取其最近一期财报的资产负债表数据，并按照港股财务字段映射规则转换为系统标准数据模型。

- 港股资产负债表字段映射：

| 系统标准字段 | 港股财报字段（IFRS/HKFRS） | 映射说明 |
|-------------|---------------------------|---------|
| cash_and_equivalents | Cash and cash equivalents / 现金及现金等价物 | 直映射 |
| total_current_assets | Total current assets / 流动资产合计 | 直映射 |
| total_assets | Total assets / 资产总计 | 直映射 |
| total_current_liabilities | Total current liabilities / 流动负债合计 | 直映射 |
| total_liabilities | Total liabilities / 负债合计 | 直映射 |
| interest_bearing_debt | Interest-bearing debt / 有息负债 | 见REQ-HK-005 |
| net_assets | Equity attributable to owners / 归属母公司股东权益 | 直映射 |

**REQ-HK-005**: When 计算港股有息负债时，the 系统应按照港股有息负债定义进行计算：有息负债 = 短期银行借款 + 长期银行借款 + 应付债券 + 其他计息负债。

- 港股有息负债可能包含A股不常见的科目（如"融资租赁负债"），需通过配置文件定义有息负债组成科目
- Where 港股财报中缺少"应付债券"字段，the 系统应将"应付债券"计为0，不视为数据缺失

**REQ-HK-006**: For 每只港股通标的股票，the 系统应获取其最近一期财报的利润表数据，并映射为系统标准模型。

- 港股利润表字段映射：

| 系统标准字段 | 港股财报字段（IFRS/HKFRS） | 映射说明 |
|-------------|---------------------------|---------|
| revenue | Revenue / 营业收入 | 直映射 |
| operating_cost | Cost of sales / 营业成本 | IFRS常用"Cost of sales"而非"营业成本" |
| net_profit | Profit for the year / 净利润 | 直映射 |
| deducted_net_profit | Profit excluding non-recurring items / 扣非净利润 | 港股无强制披露，见REQ-HK-007 |

**REQ-HK-007**: Where 港股财报未披露"扣除非经常性损益后的净利润"，the 系统应以"净利润"代替"扣非净利润"，并在结果中标注"扣非净利润为估算值"。

- 验收条件：港股无扣非净利润字段 → 使用净利润替代，标注为估算

**REQ-HK-008**: For 每只港股通标的股票，the 系统应获取其最近一期财报的现金流量表数据，并映射为系统标准模型。

- 港股现金流量表字段映射：

| 系统标准字段 | 港股财报字段（IFRS/HKFRS） | 映射说明 |
|-------------|---------------------------|---------|
| operating_cash_flow | Net cash from operating activities / 经营活动现金流净额 | 直映射 |
| capital_expenditure | Purchase of PPE / 购建固定资产支付现金 | IFRS常用"Purchase of property, plant and equipment" |
| free_cash_flow | operating_cash_flow - capital_expenditure | 计算得出 |

### 9.3 港股市值数据获取

**REQ-HK-009**: When 系统以港股通模式运行时，the 系统应获取每只港股的市值数据，包括：总市值（港币）、流通市值（港币）、最新股价（港币）。

- 数据源：akshare港股实时行情接口（如 `stock_hk_spot_em`）
- 市值单位：港币（HKD）

**REQ-HK-010**: Where 港股实时行情接口无法获取某只股票的市值数据时，the 系统应将该股票标注为"市值数据缺失"，跳过该股票不中断整体流程。

### 9.4 港股分红数据获取

**REQ-HK-011**: For 每只港股通标的股票，the 系统应获取其近3年的分红数据，包括：近3年累计每股现金分红金额、股息率、分红年数。

- 数据源：akshare港股分红接口（如 `stock_hk_dividend_detail` 或类似接口）
- 港股分红数据特点：港股以"每股派息（港仙）"为常见单位，需统一转换为"每股派息（港元）"

**REQ-HK-012**: While 港股分红数据以"仙（cent）"为单位时，the 系统应将分红金额除以100转换为港元单位，确保与港币计价的股价一致。

- 验收条件：分红数据单位为仙 → 除以100转换为港元

**REQ-HK-013**: If 港股分红数据获取失败时，the 系统应将该股票的分红数据设为默认值（0），不中断整体流程，并在日志中记录警告。

### 9.5 货币与汇率

**REQ-HK-014**: The 系统应支持港股选股结果以港币（HKD）为计价单位独立输出，不强制换算为人民币。

- 港股选股结果中所有金额指标（市值、现金流、资产垫等）均以港币计价
- 资产垫比率、资本开支率、股息率等无量纲指标不受货币单位影响

**REQ-HK-015**: Where 配置文件中启用了"汇率统一换算"选项时，the 系统应获取当前港元兑人民币汇率（HKD/CNY），并将港股市值数据换算为人民币后参与筛选。

- 默认关闭汇率换算，各市场独立计价
- 汇率数据源：akshare汇率接口或配置文件中固定汇率值
- 验收条件：配置启用汇率换算 → 港股所有金额字段乘以HKD/CNY汇率

### 9.6 港股退市风险过滤

**REQ-HK-016**: When 系统以港股通模式运行退市风险过滤时，the 系统应按照港交所退市规则进行过滤，而非A股退市规则。

- 港交所退市规则与A股不同，不使用ST标记制度

**REQ-HK-017**: The 系统应按以下港交所退市风险标准过滤港股：

| 过滤条件 | 规则说明 |
|---------|---------|
| 停牌时长过滤 | 连续停牌超过3个月的股票予以剔除 |
| 净资产为负 | 期末净资产（归属母公司股东权益）为负值的股票予以剔除 |
| 审计意见过滤 | 最近一期财报被出具"无法表示意见"或"否定意见"审计报告的股票予以剔除 |
| 最低市值过滤 | 市值低于港交所最低上市市值要求（当前为5,000万港元）的股票予以剔除 |

**REQ-HK-018**: Where 港股股票处于"退市整理期"时，the 系统应剔除该股票。

**REQ-HK-019**: If 港股退市风险过滤无法判断某只股票是否应剔除（如缺少审计意见信息）时，the 系统应保留该股票继续参与筛选，并在日志中记录警告信息。

### 9.7 核心筛选逻辑复用

**REQ-HK-020**: When 港股通选股流程进入核心筛选阶段时，the 系统应复用现有A股模块的资产垫分级逻辑（T0/T1/T2）、经营安全评估逻辑、兑现安全评估逻辑，不因市场不同而改变筛选算法。

- 资产垫分级：复用 `AssetCushionAnalyzer`，阈值可通过港股专用配置独立调整
- 经营安全评估：复用 `OperationSafetyAnalyzer`，评估标准不变
- 兑现安全评估：复用 `RedemptionSafetyAnalyzer`，评估标准不变
- 综合评分：复用 `Scorer`，权重与分值可通过港股专用配置独立调整

**REQ-HK-021**: Where 港股选股配置中定义了不同于A股的筛选阈值时，the 系统应使用港股专用阈值参与筛选，而非A股默认阈值。

- 验收条件：港股配置中T0保守比率设为0.7 → 港股T0分级使用0.7而非A股的0.8

### 9.8 结果输出

**REQ-HK-022**: When 港股通选股流程完成时，the 系统应输出港股选股结果，格式与A股选股结果一致，并附加以下港股专属字段：

- 股票代码（5位港股代码格式）
- 市场标识（"HK"）
- 货币单位（"HKD"）
- 扣非净利润是否为估算值（布尔标记）

**REQ-HK-023**: The 系统应将港股选股结果输出为独立于A股结果的CSV文件，文件名包含"hk"标识和日期，如 `hk_stock_selection_result_YYYYMMDD.csv`。

**REQ-HK-024**: When 同时运行A股和港股选股时，the 系统应分别输出A股和港股的独立结果文件，不合并为同一文件。

## 10. 配置扩展需求

**REQ-HK-025**: The 系统应在配置文件中新增"market"字段，用于指定当前选股市场，取值范围：`["A", "HK", "ALL"]`。

- `"A"`：仅运行A股（沪深300）选股
- `"HK"`：仅运行港股通选股
- `"ALL"`：依次运行A股和港股选股（默认值）

**REQ-HK-026**: The 系统应在配置文件中新增港股专用配置区块 `hk_stock`，包含以下子配置项：

```yaml
hk_stock:
  data_source:
    primary: "akshare"           # 港股数据源
    fallback: "eastmoney"        # 备选数据源
  constituents:
    include_sse: true            # 包含沪港通标的
    include_szse: true           # 包含深港通标的
  currency:
    unit: "HKD"                  # 货币单位
    convert_to_cny: false        # 是否换算为人民币
    fixed_hkd_cny_rate: null     # 固定汇率（null表示实时获取）
  delisting:
    max_suspend_months: 3        # 最大停牌月数
    min_market_cap_hkd: 50000000 # 最低市值（港元）
    filter_audit_opinion: true   # 是否过滤非标审计意见
  finance_field_mapping:         # 有息负债组成科目（可自定义）
    interest_bearing_debt_items:
      - "短期银行借款"
      - "长期银行借款"
      - "应付债券"
      - "融资租赁负债"
```

**REQ-HK-027**: Where 港股配置区块未在配置文件中定义时，the 系统应使用港股默认配置值，不因配置缺失而报错。

## 11. 主程序入口扩展需求

**REQ-HK-028**: When 主程序启动且配置中 `market` 为 `"HK"` 或 `"ALL"` 时，the 系统应初始化港股通选股流程，包括：港股通成分股获取 → 港股市值数据批量获取 → 港股财务数据逐只获取 → 港股退市风险过滤 → 核心筛选与评分 → 港股结果输出。

**REQ-HK-029**: When 主程序启动且配置中 `market` 为 `"ALL"` 时，the 系统应依次执行A股选股流程和港股通选股流程，并在日志中分别记录两个市场的运行结果。

**REQ-HK-030**: When 主程序启动且配置中 `market` 为 `"A"` 时，the 系统应仅执行A股选股流程，行为与扩展前完全一致（向后兼容）。

**REQ-HK-031**: The 主程序应在启动时输出当前选股市场标识，包括：`"A股模式"`、`"港股通模式"`、`"全市场模式"`。

## 12. 非功能需求（港股扩展）

**NFR-HK-001**: 港股通单次完整选股流程（获取数据+筛选+输出）应在8分钟内完成（港股数据源响应较慢，适当放宽）。

**NFR-HK-002**: 港股财务数据采集应支持字段映射配置化，新增或修改映射关系无需修改代码，仅需修改配置文件。

**NFR-HK-003**: When 港股数据源（akshare）接口返回字段名发生变更时，the 系统应通过配置文件中的字段映射快速适配，无需修改fetcher层代码。

**NFR-HK-004**: 港股选股模块应与A股选股模块在代码层面共享analyzer层（资产垫、经营安全、兑现安全、评分），避免逻辑重复。

**NFR-HK-005**: 系统应处理港股财报中常见的"合并报表与母公司报表差异"问题，优先使用合并报表数据。

## 14. 术语定义（港股扩展）

| 术语 | 定义 |
|------|------|
| 港股通 | 沪港通与深港通的合称，内地投资者可通过该机制买卖的港股标的 |
| 沪港通 | 上海证券交易所与香港联交所的股票互联互通机制 |
| 深港通 | 深圳证券交易所与香港联交所的股票互联互通机制 |
| HKFRS | Hong Kong Financial Reporting Standards，香港财务报告准则，基于IFRS |
| IFRS | International Financial Reporting Standards，国际财务报告准则 |
| 港仙 | 港股最小价格单位，1仙 = 0.01港元，分红数据常以仙为单位 |
| 港元/港币 | HKD，香港法定货币，港股的计价与结算货币 |
| 融资租赁负债 | IFRS下将经营租赁资本化后计入负债的科目，A股无此科目 |
| 非标审计意见 | 除"无保留意见"外的审计意见类型，包括保留意见、否定意见、无法表示意见 |
